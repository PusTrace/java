Лабораторная работа №0. Вступление

Цели работы

  1. Освоить средства компиляции приложений на Java

  2. Изучить процесс сборки исполняемых jar архивов

Используемое программное обеспечение

  1. JDK 1.5 и выше

  2. Интегрированная среда разработки на java

Справка

Компиляция

Программы на языке Java пишутся в файлах с расширением *.java и компилируются при помощи команды:

javac имя_файла

(например, javac folder/sub/Main.java)

Если главный файл связан с другими, то они тоже будут откомпилированы
После компиляции рядом со всеми файлами *.java появятся соответствующие им откомпилированные *.class файлы (один и более)

Запуск

Запустить класс на выполнение можно командой

java имя_класса

(например, java folder.sub.Main)

Для этого в нём обязательно должна присутствовать функция

public static void main(String args[])

Она же и будет стартовой точкой приложения
Расширение *.class не указывается, а / в адресе заменяются на .

Упаковка и распространение

Чаще всего для отправки нескольких связанных файлов на другой компьютер используются архивы
Java-приложение можно упаковать в архив *.jar, который к тому же возможно будет запустить без извлечения

Jar-архив может содержать ресурсы и файл манифеста META-INF/MANIFEST.MF, в котором прописывается путь к главному классу (при необходимости)
Создать jar-архив можно как вручную, так и с помощью утилиты jar из пакета JDK

Рассмотрим основные флаги утилиты jar:

Флаг Описание
c создаёт новый архив
f позволяет задать имя конечного архива
v выводит на экран информацию о каждом добавляемом файле
m позволяет указать свой файл манифеста (вместо стандартного пустого)
Пример упаковки

  • Создаём любой файл (например, MANIFEST.MF)

  • Пишем в него следующее:

Manifest-Version: 1.0

Main-Class: адрес.главного.класса.Main

Важно: в конце файла должна быть хотя бы одна пустая строка!

  • Упаковываем приложение при помощи команды:

jar -cvfm out.jar MANIFEST.MF *

Последним аргументом следуют паттерны всех добавляемых в архив файлов через пробел (здесь при помощи * добавляются все файлы из всех подпапок)

  • Готово! Получен файл out.jar. Теперь его можно запускать командой

java -jar out.jar

  • Если в манифесте нет параметра Main-Class, то для запуска необходимо дополнительно указать адрес главного класса

java -jar out.jar адрес.главного.класса.Main

Пример выполнения лабораторной работы

public class Main {

    public static void main(String[] args) {

        // Выводит в консоль надпись "Hello, world!"

        System.out.println("Hello, world!");

    }

}

Задание лабораторной работы

  1. Создать файл Main.java с программой, выводящей на экран «Hello, world!»

  2. Открыть консоль, перейти в папку с файлом Main.java и скомпилировать его при помощи команды javac Main.java

  3. Запустить приложение при помощи команды java Main и получить на экране строку «Hello, world!»

  4. Создать файл манифеста, указать в нём Main-Class: Main

  5. Упаковать всё приложение в исполняемый jar при помощи утилиты jar и проверить работоспособность через java -jar имя_архива.jar

  6. Отправить полученный jar на проверку

  1. Титульный лист

  3. Листинг программы
