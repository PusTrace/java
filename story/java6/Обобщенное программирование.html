Справка

Механизм обобщенного программирования позволяет реализовывать алгоритмы, одинаковым образом обрабатывающие различные типы данных
Обобщенные переменные записываются аналогично обычным ровно как и аргументы и тип возврата функций

Для того, чтобы объявить в классе обобщенный тип, необходимо прописать имя этого обобщенного параметра следующим образом (правила для его именования аналогичны правилам именования классов):

// T - имя первого обобщенного типа класса

public class Foo<T> {}

Несколько обобщенных параметров записываются через запятую:

public class Foo<T, A, B, SomeOther, D> {}

И, наконец, их использование:

public class Test<A, GenericType> {

    // Переменная обобщенного типа GenericType

    private GenericType variable;

    // Массив обобщенного типа A

    private A[] arr;

    // Вложенный объект класса ArrayList того же обобщенного типа

    private ArrayList<GenericType> list;



    /**

     * Функция, работающая с обобщенным типом A

     * и просто возвращающая поданное в аргументах

     * 

     * @param a возвращаемый объект

     */

    public A test(A a) {

        return a;

    }

}

Работа с данным классом:

// Создаем объект класса Test, типизированный на:

// A = String

// GenericType = Integer

Test<String, Integer> testObject1 = new Test<String, Integer>();



// Возможно сократить создание

Test<String, Integer> testObject2 = new Test<>();



// Вызов функции

String result = testObject2.test("Hello, world");



// Можно использовать любые типы

Test<Object, Float> testObject3;

Кроме того, область допустимых обобщенных типов можно ограничивать при помощи ключевых слов super (является родителем для …) и extends (является наследником для …)

interface SomeInterface {

    void someFunction();

}



class SomeClassImplementsSomeInterface implements SomeInterface {

    @Override

    public void someFunction() {

        System.out.println("Some function!");

    }

}



public class GenericBounds<T extends SomeInterface> {

    // Переменная обобщенного типа T, который ВСЕГДА наследуется

    // от SomeInterface (на любом уровне вложенности наследования)

    private T t;



    public GenericBounds(T t) {

        this.t = t;

    }



    public void test() {

        // Компилятор знает, что t типа T,

        // а T extends SomeInterface

        t.someFunction();

    }

}

SomeClassImplementsSomeInterface scisi = new SomeClassImplementsSomeInterface();

GenericBounds<SomeClassImplementsSomeInterface> obj1 = new GenericBounds<>(scisi);

В Java существует множество интерфейсов, пригодных к использованию с обобщенным программированием
Интерфейс Comparable служит для сравнения любых объектов, не поддерживающих по умолчанию численное сравнение (и одновременно информирования о том, что данный объект поддерживает это)

public interface Comparable<T> {

    /**

     * Возвращает 0, если someOtherObject равен нашему объекту

     * 1 и -1, если наш объект больше или меньше someOtherObject

     */

    public int compareTo(T someOtherObject);

}

Пример класса с использованием Comparable<T>:

public class CompareTest<E extends Comparable<E>> {

    private E value;



    public CompareTest(E value) {

        this.value = value;

    }



    public void setValue(E newValue) {

        // Если value логически больше newValue, то

        if (value.compareTo(newValue) > 0) {

            value = newValue;

        }

    }



    public E getValue() {

        return value;

    }

}

Данный класс хранит в себе объект, имеющий свойства интерфейса Comparable и переустанавливает хранящееся значение функцией setValue только в том случае, если оно больше текущего

Интерфейс Comparable реализует множество стандартных классов
Например, Integer и другие обертки над числовыми типами сравнивают значения математически, а String производит лексическое сравнение
