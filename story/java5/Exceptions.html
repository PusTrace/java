Справка

В языке Java существует механизм обработки ошибок с помощью
исключений
Если код, выполняющий некоторые операции, не может их выполнить по какой-либо причине, которую он не может или не обязан решать, то он входит в “исключительную ситуацию” и сигнализирует вызывающему коду об этом при помощи объекта класса исключения
Любое исключение содержит в своей иерархии наследования класс Throwable

Чтобы “выбросить” исключение необходимо использовать оператор throw:

// int x в аргументах функции



if (x < 0) {

    // Рекомендуется использовать наиболее подходящие по

    // смыслу классы исключений

    // В данном случае у нас неверный аргумент функции

    throw new IllegalArgumentException("x не может быть меньше нуля!");

}

Для обработки исключений служат операторы try, catch и finally:

try {

    abc(-5); // Предположим, что это функция из предыдущего примера

} catch (IllegalArgumentException iae) {

    // Возникло исключение IllegalArgumentException

    // Вызываем функцию printStackTrace()

    // которая выведет всю информацию в консоль

    iae.printStackTrace();

} finally {

    // Код, который выполнится в любом случае

    // после всех try и catch

    // Вне зависимости от возникших исключений

}

Для сигнализации о том, что функция может “выбросить” определенное исключение служит ключевое слово throws в сигнатуре функции:

void abc() throws Exception {

    throw new Exception();

}

Исключения бывают проверяемыми и не проверяемыми

  • Проверяемые исключения должны обрабатываться на этапе компиляции
    При необходимости можно добавить к сигнатуре функции оператор throws и череда обработки исключения перейдет к вызывателю этой функции

  • Не проверяемые исключения не требуют обязательной проверки и бывают двух видов:

    • RuntimeException. Наследуется от Exception и описывает потенциально возможные исключения, которые могут происходить практически во всех участках кода. Пример: NullPointerException

    • Error. Не наследуется от Exception, представляет собой отдельную ветку классов, наследующихся от Throwable. Описывает критические ошибки при работе приложения, которые обычно не позволяют приложению работать дальше. Пример: OutOfMemoryError

Пример реализации

Объявим класс исключения StackUnderflowException, которое должно
сигнализировать о том, что стек пуст, и из него невозможно извлечь ни
одного элемента

public class StackUnderflowException extends Exception {



    /**

     * @param message сообщение, хранимое в исключении

     */

    public StackUnderflowException(String message) {

        super(message);

        // Сообщение можно получить при помощи getMessage()

    }



}

“Выбросим” созданное исключение при попытке вызова peek() на пустом стеке

/**

 * @throws StackUnderflowException если стек пуст

 */

public int peek() throws StackUnderflowException {

    if (isEmpty()) {

        throw new StackUnderflowException("Стек пуст");

    } else {

        return data[top];

    }

}
