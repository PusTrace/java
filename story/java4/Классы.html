Справка

Очередь

Структура данных, организованная по принципу «первым пришел - первым ушел» (First In First Out, FIFO)
Для реализации можно использовать различные структуры данных: массив, два стека и т. д.
Независимо от реализации любая очередь должна иметь следующие методы:

  • enqueue – добавляет элемент в конец очереди

  • peek – возвращает элемент из начала очереди

  • dequeue – удаляет элемент из начала очереди

  • getSize – возвращает количество элементов в очереди

  • isEmpty – возвращает true, если в очереди нет ни одного элемента

  • isFull – возвращает true, если в очереди находится максимально
    возможное количество элементов

Поэтому очередь удобно описать интерфейсом, которого будут придерживаться конкретные реализации очередей:

public interface Queue {

    void enqueue(int element);

    int peek();

    void dequeue();

    int getSize();

    boolean isEmpty();

    boolean isFull();

}
Стек

Структура данных, организованная по принципу «последним пришел – первым ушел» (Last In First Out, LIFO)
Независимо от реализации любой стек должен иметь следующие методы:

  • push – добавляет элемент в вершину стека

  • peek – возвращает элемент из вершины стека

  • pop – удаляет элемент из вершины стека

  • getSize – возвращает количество элементов в стеке

  • isEmpty – возвращает true, если в стеке нет ни одного элемента

  • isFull – возвращает true, если в стеке находится максимально
    возможное количество элементов

Стек можно описать следующим интерфейсом:

public interface Stack {

    void push(int element);

    int peek();

    void pop();

    int getSize();

    boolean isEmpty();

    boolean isFull();

}

Пример реализации очереди

Простейшей реализацией очереди является очередь, основанная на массиве

public class ArrayQueue implements Queue {

    // Размер массива

    private final int capacity;

    // Индекс начала очереди в массиве

    private int enqueuePointer;

    // Индекс конца очереди в массиве

    private int dequeuePointer;

    // Сам массив

    private final int[] data;



    /**

     * Конструктор очереди

     * @param capacity максимальный размер очереди

     */

    public ArrayQueue(int capacity) {

        this.capacity = capacity;

        this.data = new int[capacity];

    }



    /**

     * Добавляет элемент в очередь

     * @param element добавляемый элемент

     */

    public void enqueue(int element) {

        data[enqueuePointer] = element;

        enqueuePointer = (enqueuePointer + 1) % (capacity + 1);

    }



    /**

     * @return элемент из вершины очереди

     */

    public int peek() {

        return data[dequeuePointer];

    }



    /**

     * Удаляет элемент из вершины очереди

     */

    public void dequeue() {

        dequeuePointer = (dequeuePointer + 1) % (capacity + 1);

    }



    /**

     * @return количество элементов в очереди

     */

    public int getSize() {

        int size = (enqueuePointer - dequeuePointer) % (capacity + 1);

        if (size < 0) {

            size += capacity;

        }

        return size;

    }



    /**

     * @return true, если очередь пустая

     */

    public boolean isEmpty() {

        return getSize() == 0;

    }



    /**

     * @return true, если очередь заполнена

     */

    public boolean isFull() {

        return getSize() == capacity;

    }

}

Проверка правильности работы

Для проверки корректности работы класса ArrayQueue выполним следующий фрагмент кода
Если очередь работает неправильно, программа аварийно завершится на одном из операторов assert

Queue queue = new ArrayQueue(4);

assert queue.isEmpty();

queue.enqueue(8);

assert queue.getSize() == 1;

queue.enqueue(42);

assert queue.peek() == 8;

queue.dequeue();

assert queue.peek() == 42;
