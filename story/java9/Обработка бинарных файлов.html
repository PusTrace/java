Лабораторная работа №9. Обработка бинарных файлов

Цели работы

  1. Освоить работу с бинарными файлами с использованием средств Java

Используемое программное обеспечение

  1. JDK 1.5 и выше

  2. Интегрированная среда разработки на java

Справка

Для работы с файлами в бинарном режиме в Java имеются классы FileInputStream (чтение) и FileOutputStream (запись), располагающиеся в пакете java.io

Данные классы наследуются от InputStream и OutputStream и реализовывают простейшие операции для чтения/записи одиночных байтов и массивов байтов

Для удобства чтения и записи различных типов данных существуют классы DataInputStream и DataOutputStream, которые являются оболочками поверх обычных InputStream и OutputStream, добавляющими им функционал для чтения/записи всех примитивных типов данных (int, short, byte, long, float, double, boolean, char, …), а также объектов String

Пример записи int и double в бинарный файл:

int i = 42;

double d = 3.14;

DataOutputStream dos = null;



try {

    dos = new DataOutputStream(

        new FileOutputStream("fileName.bin")

    );



    // Записываем данные

    dos.writeInt(i);

    dos.writeDouble(d);



    // Очистка внутреннего буффера и запись в файл

    dos.flush();

} catch (IOException e) {

    e.printStackTrace();

} finally {

    if (dos != null) {

        try {

            dos.close();

        } catch (Exception e) {}

    }

}

Потоки вывода в зависимости от своей реализации и обстоятельств могут в целях оптимизации буфферизовать выводимые данные и массово выводить их при переполнении буффера, при закрытии потока или по собственному желанию

Метод flush() заставляет поток вывода принудительно вывести все накопленные данные (или ничего не делать, если их нет)

XOR

Логическая операция XOR (исключающее ИЛИ) имеет следующую таблицу истинности:

Как можно заметить из таблицы, O = A xor B и в то же время A = O xor B при использовании одного и того же числа B
То есть, операция обратима

В Java для операции xor используется оператор ^
Пример использования, доказывающий обратимость на практике:

byte x = 50;

byte key = 42;

System.out.println(x); // 50



x ^= key;

System.out.println(x); // 24



x ^= key;

System.out.println(x); // 50

Существуют алгоритмы шифрования, основанные на XOR
Простейший пример чтения зашифрованного файла:

byte key = 73;



DataInputStream dis = new DataInputStream(

        new FileInputStream("encrypted.bin")

);



while (dis.available() > 0) {

    byte x = dis.readByte();

    x ^= key;



    // Используем расшифрованный байт..

}

Задание лабораторной работы

  1. Программа получает в аргументах 3 адреса файла и 1 число (ключ дешифрования)

    • args[0] - адрес исходного зашифрованного файла

    • args[1] - адрес расшифрованного файла

    • args[2] - адрес файла результата

    • args[3] - ключ дешифрования (byte). Для конвертирования в byte использовать Byte.parseByte(args[3])

  2. Написать программу, которая:

    • Расшифровывает файл args[0] ключём args[3] по алгоритму XOR

    • Записывает результат расшифровки в файл args[1]

    • Над расшифрованным файлом выполняет указанные в задании действия, записывая результат задачи в файл args[2]

  3. Проверить правильность работы программы

  4. Собрать исполняемый jar и отправить на проверку

Требования к программе

  1. Программа должна быть упакована в jar-архив

  2. jar-архив должен содержать аттрибут Main-Class в манифесте

Требования к отчёту по лабораторной работе

Отчёт должен содержать:

  1. Титульный лист

  2. Введённые команды

  3. Листинг программы

  4. Вывод
